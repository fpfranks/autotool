MANYCHAT FLOWS GUIDE — Phillips Automates
==========================================
Build these flows ONCE. They work across every connected channel automatically.
All flows use the same builder — just set the right triggers per channel.

==========================================
FLOW 1: WELCOME / MAIN MENU
==========================================
Trigger: First contact + keywords: hi, hello, hey, hiya, yo, sup, start, menu, help

Message:
"Hey! 👋 Welcome to [Business Name].
We'll get back to you straight away — what can we help you with?"

Buttons (Quick Replies):
[ 📅 Book an Appointment ]
[ 💰 Pricing ]
[ 🕐 Opening Hours ]
[ 💬 Speak to Someone ]

→ Each button links to its own flow below.

NOTES:
- Replace [Business Name] with the client's actual name
- In ManyChat: Automation → Flows → New Flow → name it "Welcome Menu"
- Set trigger: New Conversation + Keywords above


==========================================
FLOW 2: BOOKING FLOW
==========================================
Trigger: Button "📅 Book an Appointment" from Welcome Menu
         Also keywords: book, appointment, booking, reserve, slot

Step 1 — Ask for name:
"Great, let's get you booked in! 😊
What's your name?"
→ Set as User Input — save to custom field: first_name

Step 2 — Ask for number:
"Nice to meet you, {{first_name}}!
What's the best number to reach you on?"
→ Set as User Input — save to custom field: phone_number

Step 3 — Confirmation:
"Perfect! ✅
We've got your details:
📛 {{first_name}}
📞 {{phone_number}}

Someone will call you within 2 hours to confirm your booking.
Is there anything else we can help with?"

Buttons:
[ 🏠 Back to Menu ]
[ 💰 Pricing ]

Step 4 — Notify owner (Action block):
→ Add Action: Send Email Notification
To: [client's email]
Subject: New Booking Request
Body:
"New booking request from your bot!

Name: {{first_name}}
Phone: {{phone_number}}
Channel: {{channel}}
Time: {{now}}

Reply or call them to confirm."

NOTES:
- In ManyChat: add a "Send Email" action block after Step 3
- Or use "Notify by Email" in the Actions tab — enter the business owner's email
- This is what replaces Calendly — owner gets a ping, calls the customer back

==========================================
FLOW 5: DEFAULT / FALLBACK REPLY
==========================================
Trigger: All messages that don't match any other flow (set as Default Reply in ManyChat)

Message:
"Hey! 👋 Thanks for your message — we'll reply as soon as possible.

In the meantime, here's what you can do:"

Buttons:
[ 📅 Book an Appointment ]
[ 💰 See Pricing ]
[ 🕐 Opening Hours ]

Action:
→ Send email notification to owner:
"Someone messaged but the bot didn't understand.

Message: {{last_input_text}}
From: {{full_name}}
Channel: {{channel}}

Log in to ManyChat to reply manually."

NOTES:
- This catches anything the bot doesn't understand
- The owner gets notified so no message is ever truly missed
- In ManyChat: Automation → Default Reply → paste this in


==========================================
FLOW 6: SPEAK TO SOMEONE
==========================================
Trigger: Button "💬 Speak to Someone" from Welcome Menu
         Also keywords: human, person, speak, talk, agent, real person, staff, someone

Message:
"No problem at all! 😊
A member of the team will be with you shortly.

Our usual response time is under 2 hours during opening hours.

Is there anything specific you'd like to discuss so we can help faster?"

→ Set as User Input — save to custom field: query
→ Send email notification to owner with the query

Action — notify owner:
"Customer wants to speak to someone.

Name: {{full_name}}
Message: {{query}}
Channel: {{channel}}

Reply directly in ManyChat or contact them."

NOTES:
- After this flow, turn ON "Live Chat" mode in ManyChat so the owner can reply manually
- In ManyChat: Inbox → you can see and reply to all conversations here


==========================================
CONNECTING EACH CHANNEL
==========================================

INSTAGRAM:
- ManyChat → Settings → Connected Accounts → Instagram
- Must be a Professional or Creator account
- Flows work on DMs and Story Replies automatically

FACEBOOK MESSENGER:
- ManyChat → Settings → Connected Accounts → Facebook
- Must be a Facebook Page (not personal profile)
- All Messenger DMs to the Page trigger flows

WHATSAPP:
- ManyChat → Settings → Connected Accounts → WhatsApp
- Requires WhatsApp Business API (ManyChat handles this)
- Client needs a WhatsApp Business account

TIKTOK:
- ManyChat → Settings → Connected Accounts → TikTok
- Must be a TikTok Business account
- Works on TikTok DMs

SMS:
- ManyChat → Settings → Connected Accounts → SMS (via Twilio)
- You set up a Twilio number — texts to that number trigger flows
- Client forwards their business texts to the Twilio number

EMAIL:
- ManyChat → Settings → Connected Accounts → Email
- Works on inbound emails to a ManyChat-provided address
- Client sets up email forwarding

TELEGRAM:
- ManyChat → Settings → Connected Accounts → Telegram
- Create a Telegram Bot via @BotFather, connect to ManyChat
- Works on all messages to the bot


==========================================
SETTING UP A NEW CLIENT (REPEAT PROCESS)
==========================================

1. Go to ManyChat → click your account name (top left) → Create New Account
2. Name it after the client's business
3. Connect their channels (they need to give you access or you walk them through it)
4. Duplicate your demo flows into their account (ManyChat has a copy flow feature)
5. Update: business name, services, prices, hours, address, owner notification email
6. Test — send a DM to their page and make sure it replies correctly
7. Mark steps done in your dashboard → ManyChat Flows page
8. Send client the welcome email (assets/client-welcome-email.txt)

Time per client: ~20 minutes once you know the flows well.
