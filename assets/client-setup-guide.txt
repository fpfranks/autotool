========================================================
HOW CALL FORWARDING WORKS — EXPLAIN THIS TO THE CLIENT
========================================================

Call forwarding only activates when you don't answer. Your phone rings normally
for 15–20 seconds. If you pick up — great, business as usual. If you can't answer,
the call automatically diverts to your AI receptionist instead of going to voicemail.

You don't change your existing business number. Callers still ring the same number
they always have. The only difference is: now someone always answers.

To turn it off at any time, dial the cancel code below. It's instant.

────────────────────────────
Landline / VoIP number
────────────────────────────
Landlines and VoIP numbers cannot use USSD codes. The client needs to:
1. Log in to their phone provider's online account, OR
2. Call their provider and ask them to set up "conditional call divert"
   to the AI number when calls go unanswered.

Most providers (BT, Virgin, Vonage, etc.) offer this as a free setting.
It takes them about 10 minutes with their provider on the phone.

Send them this message:
"Hi [NAME]! Your AI receptionist is ready. Because you have a landline/VoIP number,
you'll need to set up call forwarding through your phone provider rather than dialling
a code. Just ring them and ask to set up 'conditional call divert' to [AI NUMBER] —
it's a standard free feature and takes about 10 minutes. Let me know when it's done
and I'll test it from my end! — Frankie, Phillips Automates"

========================================================
TROUBLESHOOTING — COMMON CLIENT ISSUES
========================================================

"The codes aren't working"
→ Check they're dialling correctly — no spaces, press Call after each
→ Some phones need you to hold # until it connects — tell them to try that
→ Ask what network they're on — different networks use slightly different codes
→ EE: confirm the number starts with +44, not 07

"It's going straight to voicemail still"
→ The no-answer code sets a ring time. 20 seconds is default. Check code was dialled correctly.
→ Ask them to test by calling from another phone and deliberately not answering

"The AI isn't answering"
→ Check the Telnyx number is connected to the VAPI assistant in the VAPI dashboard
→ Check VAPI assistant is not paused or over usage limits
→ Call the Telnyx number directly (not via forwarding) — if AI answers, the issue is the forwarding code
→ If AI doesn't answer direct, the VAPI/Telnyx connection needs checking

"The AI said something wrong"
→ Update the VAPI system prompt with the corrected info
→ Changes take effect immediately
→ Reassure client — just email Frankie with what needs changing, done within 24 hours
